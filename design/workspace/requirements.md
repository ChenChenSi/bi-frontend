# 工作区管理需求文档

## 1. 概述
工作区 (Workspace) 是 BI 系统中用于组织和隔离资源的顶层容器。为了便于多团队协作，系统支持按工作区对报告进行分组管理。

## 2. 功能需求

### 2.1 工作区管理
- **创建工作区**: 用户可以创建新的工作区，用于存放特定业务线或团队的报表。
- **列表展示**: 用户登录后，可以看到自己所属的所有工作区列表。
- **元信息设置**: 工作区管理员可以修改工作区的名称、描述等基本信息。

### 2.2 成员与角色管理
每个工作区内拥有独立的成员和角色体系：

1.  **工作区管理员 (Workspace Admin)**:
    - **定义**: 当前工作区的超级管理员。
    - **权限**:
        - 设置工作区元信息。
        - 管理工作区成员（添加、移除）。
        - 分配角色：可以指定其他成员为“工作区管理员”、“报告管理员”、“报告编辑员”或“访客”。
        - 管理用户群组：创建群组以便批量授权。
        - 接收所有重要的系统通知（如权限申请抄送）。

2.  **报告管理员 (Report Admin)**:
    - **定义**: 特定报告的负责人。
    - **权限**:
        - 创建、编辑、删除其负责的报告。
        - 管理该报告的共享权限。
        - 接收该报告的权限申请通知。
        - 接收该报告组件加载失败的告警通知。

3.  **报告编辑者 (Report Editor)**:
    - **定义**: 参与报告设计的成员。
    - **权限**: 进入画布编辑报告内容、配置图表、保存设计。

4.  **工作区访客 (Visitor)**:
    - **定义**: 仅需查看报告的成员。
    - **权限**: 仅具备只读阅览权限，无法进入编辑模式。

### 2.3 用户群组
- 支持创建用户组（如“销售部”、“财务部”），便于在分享报告时进行批量授权。
